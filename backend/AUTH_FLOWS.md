## Summary

### Key Points

1. **Two User Types:**
   - SUPER_ADMIN: Global access, no organization
   - ORG_USER: Scoped to organization

2. **JWT Payload:**
   - Contains userId, email, role, organizationId
   - organizationId is null for SUPER_ADMIN

3. **Middleware Chain:**
   - authenticate â†’ Validates JWT
   - authorize â†’ Checks role
   - tenantContext â†’ Validates organization
   - requireOrganization â†’ Ensures org membership
   - requireSuperAdmin â†’ Ensures SUPER_ADMIN

4. **Data Isolation:**
   - SUPER_ADMIN: No filter, access all data
   - ORG_USER: Filter by organizationId

5. **Security:**
   - Password hashing with bcrypt
   - JWT signing and verification
   - Organization status validation
   - User status validation
